Edit Save Partition

Once you have made changes to the save partition in your file manager, you must hit enter in the console window when prompted to close the partition and complete the process.

Allows editing of the contents of a UCE's save partition in your OS's file manager.

Inputs

  • Input Path: Required The UCE file whose save partition you want to edit.
  • Mount Method: Optional, Linux only, needs root. If selected, the save partition will be mounted as with any other Linux filesystem rather than edited using a combination of debugfs/make2fs commands. There ought to be no practical difference; the mount method is mostly a hold-over from early attempts to implement this functionality. A small advantage is that you don't have to hit enter in the console window to complete the process. A big disadvantage is that you'll need another file manager window opened as root to drag/copy/paste files in and out of the save partition.
  • File Manager Optional, Linux only. Specify a particular file manager application. Otherwise the tool works through a list of known file managers until it finds one. The list is not exhaustive so if you are using something niche you may need to specify it explicitly.
  • Backup Uce: Optional Select this option to create a backup of the UCE before any write operations happen. It will be created in the same directory as the UCE with the extension '.bak'.
