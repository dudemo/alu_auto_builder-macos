Scrape Metadata and Create Recipes

This operation requires administrator privileges on Windows

Provide a directory containing roms and produce 'recipe folders'. The tool uses the excellent Skyscraper under the hood to do download information and covers for the roms.

This operation is useful where you have a set of roms which you want to turn into UCEs but have some need to tweak the contents of the UCEs before they are built. This usually means applying a custom save partition. If you don't care about this and just want UCEs, use 'Scrape To UCEs' from the main menu.

Inputs

  • Input Dir: Required A directory containing one/multiple roms for a single emulated system.
  • Output Dir: Optional The directory in which a recipe dir for each rom will be placed. If one isn't provided it will be created in the same directory as the roms.
  • Core Path: Required The path to a libretro core (emulator) file compatible with the roms in the input directory.
  • Bios Dir Optional Any files in this directory will be placed in the 'roms' directory within the UCE. This is mostly useful for adding the same bios file(s) to all recipes.
  • Platform Required The platform/system the roms belong to (e.g. for MAME2003 choose 'mame-libretro'). Just because a platform is listed here it doesn't mean an appropriate core exists. It's been left on the long side as a modest nod to future-proofing.
  • Scrape Module Required The online source from which to scrape metadata. Each has a website if you need to know more. Some (e.g. the superb screenscraper) work a lot faster with a user account. arcadedb is excellent, requires no account but as the name suggests is relevant mostly for arcade roms.
  • User Name Optional A user name if you are using a module for which you have an account.
  • Password Optional A password for any user account.
  • Refresh Rom Data Optional Select to re-download metadata and images for input roms. Do this if the tool is not producing a folder of marquees as expected.
