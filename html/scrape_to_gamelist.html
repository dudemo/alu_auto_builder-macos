Scrape Metadata and Create Gamelist

Provide a directory containing roms and produce an EmulationStation compatible gamelist.xml (the sort used by RetroPie amongst others). Only the data required to produce UCEs (titles, descriptions, covers) will be downloaded. The tool uses the excellent Skyscraper under the hood to do this.

This is functionality is provided because it's a step in the process from getting from a directory of roms to either recipes or UCEs, and it was straightforward to break it out. If you are interested in building gamelists, especially for uses other than building UCEs, then using either Skyscraper itself or a GUI tool like Skraper is probably a better option. Both provide much more control over what is downloaded and how it is formatted.

Inputs

  • Input Dir: Required A directory containing one/multiple roms for a single emulated system.
  • Output Dir: Optional The directory in which the gamelist.xml and a 'media' directory containing downloaded covers will be placed. If one isn't provided it will be created in the same directory as the roms.
  • Platform Required The platform/system the roms belong to (e.g. for MAME2003 choose 'mame-libretro'). Just because a platform is listed here it doesn't mean an appropriate core exists. It's been left on the long side as a modest nod to future-proofing.
  • Scrape Module Required The online source from which to scrape metadata. Each has a website if you need to know more. Some (e.g. the superb screenscraper) work a lot faster with a user account. arcadedb is excellent, requires no account but as the name suggests is relevant mostly for arcade roms.
  • User Name Optional A user name if you are using a module for which you have an account.
  • Password Optional A password for any user account.
